Event Lifecycle

Three Phases

Every event follows the same process: prepare, track, follow up.

1-4 weeks before the event

Pre-Event: Setup & Outreach

Set up the event workspace, identify targets, and begin outreach.

  • HubSpot Manager creates the event Google Sheet from template and imports the full attendee list
  • HubSpot Manager creates the #event-[name] Slack channel and configures automation workflows
  • SDRs browse the attendee list and mark priority targets (High / Medium / Low)
  • SDRs reach out via event app, email, or LinkedIn — update outreach status in the Sheet
  • Contacts who respond or book a meeting get imported to HubSpot with the event as lead source
During the event

During Event: Live Tracking

Reps post updates in Slack. AI parses and structures the data. Leadership monitors the dashboard.

  • Sales Reps post free-form updates in the event Slack channel after every interaction
  • AI parses each message → extracts company, contact, status, interest level, and notes
  • AI confirms what it captured in a Slack thread reply (reps can correct if needed)
  • Data syncs from Google Sheet to HubSpot every 15-30 minutes automatically
  • Leadership monitors the live Summary Dashboard tab and HubSpot reports
Within 1 week after

Post-Event: Follow-Up & Reporting

System generates follow-up recommendations. Team reviews and approves before execution.

  • System runs final sync and generates a "Follow-Up Review" tab with proposed actions per company
  • AEs & Reps review the follow-up list and approve or adjust actions
  • Approved hot leads get enrolled in HubSpot sequences; tasks created for all tiers
  • Team gets a summary in Slack: companies met, hot leads, follow-up status
  • Marketing pulls event ROI data from HubSpot reports (lead source = event name)
For Sales Reps

How Reps Log Meetings

After every meeting or interaction, post an update in the event Slack channel. The AI parses your message into structured data.

# event-manifest-2026

AI
Event Tracker 2:15 PM
Captured 2 interactions at LogiCorp:
• Sarah Chen (VP Supply Chain) — Completed / Hot — wants demo next week
• Mike (CTO) — Booth Visit / Warm — seemed interested
Reply to correct anything.
How it works Write naturally — the AI extracts the company, contact, status, and interest level. It confirms in a thread reply so you can correct anything before it syncs to the Sheet and HubSpot. If your message is missing details (e.g., contact name or interest level), the AI captures what it can and asks for the rest in the thread.

The Review & Enrollment Flow

Generator runs
→
Follow-Up Review tab
→
AE reviews
→
Enrollment or task
  1. HubSpot Manager runs the post-event follow-up generator workflow
  2. System creates a "Follow-Up Review" tab in the event Google Sheet with one row per company: contact name, interest level, notes, recommended sequence, and an Action column
  3. AEs/Reps review each row and set the Action to either "Approve" or "I'll Handle"
  4. System picks up actions every 15 minutes:
    • Approve → contact is enrolled in the recommended sequence automatically
    • I'll Handle → a HubSpot task is created and assigned to the rep (they enroll manually or use a custom sequence)
  5. follow_up_status is updated on both the Sheet and the Event Interaction record in HubSpot
Nothing fires without review No sequences are triggered automatically. The rep must explicitly set "Approve" or "I'll Handle" for each company before anything happens.

Lifecycle Stages

When Does a Contact Move to MQL or SQL?

Lifecycle stage progression is tied to meeting status, not just intent.

Meeting StatusLifecycle StageRationale
Meeting scheduled (before event)MQLIntent shown by booking, but no conversation yet
Meeting completedSQLActual conversation occurred — qualified by sales
No-showMQLMeeting was booked but didn't happen — stays MQL
Booth visit / Walk-upSQLDirect interaction happened — treated as sales-qualified
Key rule A contact becomes an MQL when a meeting is scheduled. They only move to SQL when the meeting actually happens. If a contact is already at a later stage (e.g., Opportunity), the event interaction does not downgrade them.

Multi-Event Support

What Happens When a Contact Attends Multiple Events?

Contacts and companies often appear at multiple conferences. Each event interaction is stored as a separate HubSpot custom object record — no data is overwritten.

The problem with contact properties If event data were stored as contact properties, attending a second event would overwrite the first event's data. Sarah Chen's "Hot" interest at Manifest would be replaced by her "Warm" rating at ProMat. All historical data lost.
The solution: "Event Interaction" custom object Each event touchpoint is its own record in HubSpot, linked to both the contact and the company. A contact can have 5 Event Interactions across 5 events — full history preserved, fully reportable.

Associations

↔
Contact

Many Event Interactions can link to one Contact. Many Event Interactions can link to one Company. This enables reporting like: "Show me all Hot interactions from Manifest 2026" or "Which companies have we met at 2+ events?"

Contact-level property: lead_source The contact's lead_source is still set to the event name for the first event where they were sourced. This preserves lead attribution. The event_outreach_status property on the contact tracks pre-event outreach via HubSpot sequences.
